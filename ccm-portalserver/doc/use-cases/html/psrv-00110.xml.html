Use Case 00110:Configure A Portal Tab

Release Target: 2.0

Description:

Tabs are the principle organizational mechanism for restricting view clutter and focusing Portal data and function. This Use Case describes the procedure to follow when a modification to a Tab is desired.

Goal Level: summary_goal
Actors:
  • Portal Administrator
Trigger: A PA desires to modify the configuration for a Portal Tab
Preconditions:

Basic Path:


  1. Starting at the Portal, he PA selects the "Configure Portal" link.
  2. The 'Layout' tab is chosen next.
  3. At the Configure Portal page, the PA is presented a list of all Tabs currently residing within the Portal. The PA selects a Tab to configure.
  4. The PA makes the desired configuration changes and then clicks the "Save Configuration" button. Configuration changes available include:
    • Change the layout of portlets (PSRV-00100:Add a Tab to a Portal)
    • Rename a Portal Tab (PSRV-00140:Change Name of Existing Portal Tab
    • Delete a Portal Tab (PSRV-00120:Delete a Portal Tab)
    • Re-order tabs in a Portal (PSRV-00150:Reorder Portal Tabs)
    • Add and Delete Portlets from a Portal (PSRV-00200:Add a Portlet to a Portal Tab PSRV-00210:Delete a Portlet from a Portal Tab)
Postconditions:

Succesful End Conditions:

The desired config changes are made and saved.
