Use Case 00910:Create a Theme for a Portal

Release Target: 2.0

Description:
Goal Level: user_goal
Actors:
  • Portal Administrator
Trigger: A desired Look and Feel is not available in the list of currently installed themes.
Preconditions:
Basic Path:
  1. The PA clicks the 'Create Theme' button on the Themes tab in the Configure Portal area
  2. The PA is taken to the theme creation page
  3. The currently selected theme is pre-loaded into the UI, so that PA's can easily make incremental changes.
  4. The PA creates the theme. Individual aspects of this process are detailed in use cases psrv-00950 through psrv-00970
  5. When satisfied, the PA clicks the 'Save Theme' button.
  6. The UI prompts the PA for a name for the theme. After entering one, the PA clicks 'Save' and the PA is returned to the Configure Portal Themes tab, where the new theme appears in the list.
